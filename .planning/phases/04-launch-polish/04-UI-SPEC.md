## Spacing Scale

Standard 8-point scale in use across the project. Phase 4 does not add
new spacing tokens. No JS tailwind config — all tokens live in
`src/app/globals.css` under `@theme`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, grid gap-1 between habit cells |
| sm | 8px | Compact element spacing, gap-2 in right cluster |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, card p-6 |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Phase 4 exception — touch targets:**
All interactive cells and icon buttons in this phase must reach 44px
(h-11 w-11 = 44px) regardless of the standard 40px `size="icon"` default.
This is a targeted override, not a new scale token.

- HabitGrid cells: `h-11 w-11` (44px) — unconditional, all breakpoints
- CountCard stepper ±buttons: `h-11 w-11` (44px) — overrides `size="icon"` default
- CountCard Apply button: `h-9` minimum (36px is acceptable for secondary action that only appears when stepperValue !== 0)
- MonthNavigator right cluster buttons: `size="icon"` (40px) — acceptable because these are navigation controls, not progress-logging targets

Source: CONTEXT.md D-03, D-04

## Color

All tokens are defined in `src/app/globals.css` under `@theme`. Dark mode
overrides via `@media (prefers-color-scheme: dark)` — no toggle in v1.
Phase 4 adds zero new color tokens.

| Role | Light mode token | Dark mode token | Usage |
|------|-----------------|-----------------|-------|
| Dominant (60%) | `--color-background` oklch(99% 0 0) zinc-50 | oklch(11% 0.005 260) zinc-950 | Page background, full-bleed surface |
| Secondary (30%) | `--color-card` oklch(100% 0 0) white | oklch(15% 0.006 265) zinc-900 | All goal cards, dialog surfaces, nav |
| Accent (10%) | `--color-primary` oklch(54% 0.155 161) emerald-600 | oklch(60% 0.146 162) emerald-500 | Reserved elements only (see below) |
| Destructive | `--color-destructive` oklch(58% 0.215 27) red-600 | oklch(65% 0.210 25) red-500 | Delete confirmation, error alerts only |

**Accent (primary emerald) reserved strictly for:**
- Progress bar fill on all goal cards
- Hit habit grid cells (bg-primary)
- +1 count button and primary CTAs (bg-primary)
- Today ring on habit grid (ring-ring)
- Focus rings (--color-ring emerald-400)

**Accent is NOT used for:** navigation icons, right cluster buttons,
stepper buttons, toast backgrounds, or any state that is neutral/informational.

**Toast error color:** Sonner's built-in destructive styling applies
`--color-destructive` automatically. No custom color override needed.

Source: globals.css (Phase 1 + Phase 2 tokens), CONTEXT.md D-05

## Component Inventory

Components modified in Phase 4 — what changes and what stays the same.

### MonthNavigator (`src/components/month-navigator.tsx`)

**Change:** Right cluster buttons become responsive icon-only on mobile,
icon + label at `md:` breakpoint.

| Button | Icon | Mobile label | Desktop label (md:) | aria-label | title |
|--------|------|-------------|---------------------|-----------|-------|
| Today | `CalendarCheck` (lucide) | hidden | "Today" | "Return to this month" | "Return to this month" |
| New Goal | `Plus` (already used) | hidden | "New goal" | "New goal" | "New goal" |
| Log out | `LogOut` (lucide) | hidden | "Log out" | "Log out" | "Log out" |

**Pattern:** Wrap label text in `<span className="hidden md:inline">` adjacent
to the icon. Icon is always visible. This keeps the `rightCluster` ReactNode
slot unchanged — RSC `[month]/page.tsx` assembles it.

**Existing pattern to follow:** ChevronLeft/Right already use
`aria-label="Previous month"` on ghost icon buttons.
Right cluster must match this contract exactly.

**No stacking, no dropdown.** Single header row at all breakpoints.

Source: CONTEXT.md D-01, D-02

### HabitGrid (`src/components/habit-grid.tsx`)

**Change:** Cell className `h-9 w-9` → `h-11 w-11` (36px → 44px).
Apply unconditionally — no responsive class, no breakpoint.

**Math verification:** 7 × 44px cells + 6 × 4px gaps (gap-1) = 332px.
Available at 375px viewport: 375 − 2 × 16px padding = 343px. Fits with 11px spare.

**Header row:** Day label cells (`S M T W T F S`) use plain `<div>` elements
inside `grid grid-cols-7 gap-1`. They auto-size to column width. No explicit
width change needed on the header row.

Source: CONTEXT.md D-03

### CountCard (`src/components/goal-card/count.tsx`)

**Change:** Stepper ± icon buttons get `className="h-11 w-11"` added.
Current code uses `size="icon"` (shadcn default = 40px h-10 w-10).
Override with explicit Tailwind: `size="icon" className="h-11 w-11"`.

**Apply button:** Stays `size="sm"` but add `className="h-9"` minimum.
Only renders when `stepperValue !== 0` — bounded visual impact.

**+1 button:** Already `className="h-11"` — no change needed.

Source: CONTEXT.md D-04

### ReflectionCard (`src/components/reflection-card.tsx`)

**Change:** Replace the inline `Alert` error display in the `else` branch of
the autosave result with a `toast.error()` call. The textarea input is
preserved in RHF state — no special preservation logic needed.

**Current code path:**
```
if (result.ok) { setSavedAt(...); setSaveError(null) }
else { setSaveError(result.error) }
```

**Phase 4 code path:**
```
if (result.ok) { setSavedAt(...) }
else { toast.error('Reflection not saved — check your connection') }
```

Remove the `saveError` state and the `Alert` block entirely.
The `toast` import is already available from `sonner` (used in dashboard-shell).

Source: CONTEXT.md D-05

### Auth Forms (`src/app/(auth)/`)

**Scope:** Audit pass — not a redesign. Check all four surfaces:
login-form, signup-form, reset-password-form, update-password-form.

**Contract per form:**
1. Server action returns `{ ok: false; error: string }` → error displayed via
   `setError("root", { message })` + `<Alert variant="destructive">` with
   `aria-live="polite"` wrapper.
2. Input field values are preserved in RHF state on failure (uncontrolled inputs = yes).
3. No silent failure path: every `result?.error` branch is handled, no early
   return without surfacing the error.

**Login form status:** Already compliant — `errors.root?.message` displayed
in `<Alert variant="destructive">` with `aria-live="polite">` wrapper.
Audit the other three forms for same coverage.

Source: CONTEXT.md D-06

## Interaction Contracts

### Responsive Right Cluster

- Viewport < 768px (below `md:`): icons only, no visible text
- Viewport ≥ 768px: icon + text label side by side
- Transition: CSS `hidden md:inline` on the `<span>` containing the label text
- No animation, no layout shift — just display:none / display:inline
- Touch targets: `size="icon"` buttons (40px) — acceptable for navigation buttons
  (POLSH-01 requires 44px for progress-logging controls specifically)

### HabitGrid Touch Targets

- All cells: 44px × 44px (`h-11 w-11`)
- Interaction states remain unchanged: hit/miss/today/future color logic untouched
- Grid gap stays `gap-1` (4px)
- ARIA contract unchanged (role="grid", role="gridcell", aria-pressed, aria-label)

### CountCard Stepper

- Minus button: `h-11 w-11` (44px)
- Plus button: `h-11 w-11` (44px)
- Apply button: `h-9` minimum (36px) — only visible when stepperValue !== 0
- +1 button: `h-11` (already correct — no change)

### Reflection Autosave Error

- Trigger: `upsertReflectionAction` returns `{ ok: false }` OR network throws
- Response: `toast.error('Reflection not saved — check your connection')`
- Input state: preserved in RHF field — user can retype to trigger another save
- No retry button in toast — next keystroke triggers the next autosave naturally
- Remove the existing inline `Alert` error block from ReflectionCard

### Production Deploy Verification Checklist

These are human-action checkpoints, not UI components:

| Check | Method | Pass criteria |
|-------|--------|---------------|
| Cookie flags | Browser DevTools → Application → Cookies → sb-* | Secure + HttpOnly + SameSite=Lax all present |
| Lighthouse CLS | Chrome DevTools Lighthouse on live Vercel URL | CLS < 0.1 |
| Supabase rate limiting | Supabase dashboard → Authentication → Settings | "Enable rate limiting" confirmed active |
| Smoke test | Manual login → create goal → log progress on live URL | No errors, session persists |
