# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Month Navigation, History & Reflection.
> Generated by gsd-ui-researcher. Consumed by gsd-ui-checker, gsd-planner, gsd-executor, gsd-ui-auditor.
>
> **Inherits Phase 1 + Phase 2 foundation verbatim** (Geist font, `@theme` tokens in `src/app/globals.css`,
> Tailwind v4 default spacing scale, 4-size / 2-weight typography, 60/30/10 zinc+emerald+amber palette,
> 44px touch targets, `prefers-color-scheme` dark mode, focus-visible emerald-400 2px ring + 2px offset,
> progress-bar `scaleX` motion spring, habit grid, pace chip, create/edit/delete dialogs, undo toast).
>
> This spec EXTENDS Phase 2 with four new components:
> 1. `MonthNavigator` — `← [Month Year] →` header with optional `Today` button + keyboard nav
> 2. `WelcomeToMonth` — Copy-from-last-month / Start-fresh card on empty returning-user months
> 3. `PastEmptyState` — minimal "No goals in [Month Year]" + back-to-current deep-link fallback
> 4. `ReflectionCard` — two-textarea card with 280-char live counter + debounced autosave
>
> Plus a **read-only variant** of Phase 2's goal cards (same anatomy, zero interactive affordances,
> no kebab, no optimistic wiring, no undo toast mount — this is the portfolio-of-wins presentation).
>
> **Load-bearing product rule (PITFALLS §1 + §4):** past-month UI must never punish. No red X on
> historical habit grids. No "you missed N days last month" copy anywhere. Welcome prompt replaces
> the silent-rollover anti-pattern with an intentional moment. Past months read frozen and proud.

## Brand Tokens — No New Tokens in Phase 3

Phase 3 introduces **zero new `@theme` tokens**. Every visual surface in Phase 3 (month nav header, Welcome card, PastEmptyState, ReflectionCard, past-month frozen cards) composes from the Phase 1 + Phase 2 palette.

The STATE.md Plan 01-05 lesson still holds and is restated here so the executor cannot regress:

1. Do NOT define `--spacing-xs`, `--spacing-sm`, `--spacing-md`, `--spacing-lg`, `--spacing-xl`, `--spacing-2xl`, `--spacing-3xl` — they shadow Tailwind v4 core and break `max-w-*`, `w-*`, `p-*`, `gap-*`, `space-*`.
2. Do NOT rename existing `--color-*` tokens.
3. Do NOT add `--radius-*` tokens — Tailwind v4 ships `rounded-sm` / `md` / `lg` / `xl` / `2xl` / `full`.
4. For one-off sizing (character-counter text width, textarea min-height), use arbitrary values (`min-h-[88px]`).

Token reuse inventory for Phase 3 surfaces:

| Surface | Token | Role |
|---------|-------|------|
| MonthNavigator text ("April 2026") | `--color-foreground` | Same as Phase 2 page title |

## Spacing Scale

Inherited from Phase 1 / Phase 2 unchanged. All multiples of 4 from Tailwind v4's default `--spacing-*` utility set.

| Token | Value | Phase 3 Usage |
|-------|-------|---------------|
| `p-1` / `gap-1` | 4px | Icon-to-text gap inside Welcome card bullet list (if used), char-counter internal padding |
| `p-2` / `gap-2` | 8px | Month-nav prev/title/next inner gap, reflection header-to-body rhythm, "Saved" indicator padding |
| `p-3` / `gap-3` | 12px | Reflection textarea-to-label rhythm, Welcome card button-to-button gap |
| `p-4` / `gap-4` | 16px | Welcome card inner padding (horizontal), ReflectionCard inner padding (horizontal), gap between month-nav header and first goal card (= same as inter-card gap) |
| `p-6` / `gap-6` | 24px | Welcome card vertical padding, ReflectionCard vertical padding, PastEmptyState section rhythm |
| `p-8` | 32px | PastEmptyState top padding when rendered alone in the viewport |
| `p-12` | 48px | (unused in Phase 3) |

Exceptions:

- **Touch targets: ≥ 44px** on every new interactive element — prev/next nav arrows, Today button, Welcome Copy/Fresh buttons, PastEmptyState back button. Reflection textareas already exceed this naturally (`min-h-[88px]`).
- **Month-nav arrow buttons:** shadcn Button `variant="ghost" size="icon"` — 44×44px touch area, lucide `ChevronLeft` / `ChevronRight` at 16px.
- **Reflection textarea minimum height:** `min-h-[88px]` (fits ~three lines of 14px text at 1.5 line-height + textarea padding). Honors the "two-line" spirit of POLSH-04 without constraining the user to literally two lines.
- **Reflection textarea max height:** `max-h-[240px]` with `resize-none` — beyond this length the textarea scrolls internally. 280 chars typically renders in ~4–5 lines so this cap is rarely hit.
- **Character-counter line:** `mt-1` (4px above the counter, under the textarea). Counter sits right-aligned.

Inherited from Phase 1 / Phase 2 unchanged. Phase 3 introduces **zero new sizes and zero new weights**.

No new typography tokens introduced. Still exactly 4 sizes, exactly 2 weights — Phase 1/2 contract preserved.

### Accent Reserved-For List — Extended from Phase 2

`--color-primary` (emerald-600) is used in Phase 3 ONLY for:

1. **Past-month historical progress-bar fill** — same visual as current-month bars. A completed goal from March still renders its emerald bar proudly; greying it out would kill the "portfolio of wins" framing.
2. **Past-month historical habit-grid hit cells** — same emerald fill as the current month. Miss cells stay `bg-muted`, never red.
3. **Welcome "Copy from last month" button** — the primary forward-action on a first-of-month view.
4. **Focus-visible ring on all new Phase 3 interactive elements** (via `--color-ring` emerald-400, lighter sibling).

`--color-primary` is NOT used in Phase 3 for:

- Month-nav arrow buttons (`variant="ghost"`, neutral) — arrows are chrome, not accent.
- Today button (`variant="outline"` or `ghost`, neutral).
- Welcome "Start fresh" button (`variant="outline"`) — deliberately NOT accent so the two buttons have different visual weights.
- Reflection "Saved" indicator — uses `--color-success-muted` (Phase 2 pace-ahead tokens).
- PastEmptyState "Back to current month" button (`variant="outline"`) — it's an escape hatch, not a CTA.
- Reflection textarea borders / backgrounds (neutral input chrome).

### Destructive Reserved-For List — Phase 3 Extension

`--color-destructive` in Phase 3 is used ONLY for:

1. Reflection character counter text when count reaches 280 (the hard-block threshold). Text color only — no red background, no red border, no shake animation.
2. Autosave-failure inline `<Alert>` (baseline; Phase 4 hardens).

`--color-destructive` is NEVER used for:

- Past-month visual indication (past is celebrated, not flagged).
- Past-month "this is read-only" messaging — the affordance absence IS the signal. No lock icon, no red banner, no "Read-only" pill.
- Welcome "Start fresh" button — starting fresh is an action, not a deletion.
- Month-nav disabled-arrow state (uses `opacity-50` + `muted-foreground`, not destructive).
- PastEmptyState — no goals ≠ error.
- Character counter at the 250–279 amber band (that's warning, not destructive).

### 60 / 30 / 10 Split Verification — Phase 3 Surfaces

- **60% background:** Dashboard page `bg-background`. On Phase 3 pages (past/current/future/past-empty) the viewport remains overwhelmingly this neutral surface.
- **30% surface:** Goal cards (unchanged), MonthNavigator row (`bg-background` with `hover:bg-muted` on arrows), Welcome card (`bg-card`), Reflection card (`bg-card`), PastEmptyState card (`bg-card`). Together these occupy ~30% of the painted surface above the fold.
- **10% accent:** Progress-bar fills (current + past months), habit hit-cells (current + past months), Welcome "Copy from last month" button. The parts that DRAW THE EYE. On a past-month view the accent DOES appear on historical progress bars — this is intentional and product-rule-locked (PITFALLS §1).

## Copywriting Contract

### MonthNavigator (new header — replaces Phase 2's static page title)

| Element | Copy | Notes |
|---------|------|-------|
| Month title (`<h1>`) | `{Month Year}` | `format(viewedMonth, "MMMM yyyy")` — e.g. "April 2026" (LLM reminder: today is 2026-04-20) |
| Prev-arrow `aria-label` | Previous month | Always enabled per D-05 (history is unbounded) |
| Next-arrow `aria-label` (enabled) | Next month | Enabled when `viewedMonth < current + 1` |
| Next-arrow `aria-label` (disabled, at current+1 cap) | Next month — unavailable | Matches `aria-disabled="true"`; also serves as a `title` tooltip |
| Today button | Today | Shown only when `compareMonth(viewed, current) !== 'current'` (D-08) |
| Today button `aria-label` | Return to this month | Clearer than just "Today" for assistive tech |

The three elements render in a single flex row. Logout lives where Phase 2 placed it (right side); Phase 3 slots the Today button between the next-arrow and the pre-existing "New goal" / "Log out" buttons. "New goal" is hidden on past-month routes (D-12 layered defense → no mutation surface).

### Dashboard Header — Past-Month Variant

On past-month routes, the Phase 2 "New goal" button is **not rendered**. The right cluster is just `[Today] [Log out]`. No "Read-only" pill, no lock icon — the absence of "New goal" is itself the signal (PITFALLS §1 unpunitive framing).

### Welcome to [Month] Card (rendered per D-18)

| Element | Copy |
|---------|------|
| Headline (`<h2>`, 24px semibold) | Welcome to {Month Year}. |
| Body (16px regular) | Carry forward from {Prior Month} or start fresh? |
| Prior-month variable | `format(priorMonth, "MMMM")` — bare month name, no year (e.g. "March" when viewing April 2026) |
| Primary button | Copy from last month |
| Primary button loading | (Loader2 spinner — no text) |
| Secondary button | Start fresh |

When the copy completes (server action resolves `{ok: true}`) the Welcome card unmounts and the newly-copied goal cards appear in its place. The Phase 2 `AnimatePresence + layout` pattern on DashboardShell handles the insertion animation.

Trigger precondition (D-18):

1. Viewed month is current OR next-future, AND
2. Viewed month has zero goals for this user, AND
3. The immediately prior month has ≥ 1 goal for this user.

If the user has zero prior-month goals (first-time user / first-ever month), fall back to Phase 2's existing `EmptyState` component instead (the "It's April 2026. What do you want to commit to?" empty state with 3 greyed example cards + "Add your first goal" CTA). NO fork of the empty-state copy per tier.

### Welcome Card Interactive Behavior

- **"Copy from last month" click:** Calls `copyGoalsFromLastMonthAction` server action. Button enters loading state (Loader2). On `{ok: true}`, page revalidates and the new cards render in place of the Welcome. On `{ok: false}`, inline `<Alert variant="destructive">` with copy "Couldn't copy last month's goals. Please try again." — the Welcome card stays visible so the user can retry.
- **"Start fresh" click:** Dismisses the Welcome card for the current page session (React state, no DB flag — D-20). Falls back to Phase 2's existing empty state (headline "It's {Month} {Year}. What do you want to commit to?" + 3 greyed examples + "Add your first goal" CTA). The "Add your first goal" CTA opens the existing `CreateGoalDialog` unchanged.

### PastEmptyState Card (D-16)

Rendered when the viewed month is in the past (per `compareMonth === 'past'`) AND there are zero goals for that month. Typical trigger: user deep-links to `/dashboard/2025-12` when they signed up in 2026-04.

| Element | Copy |
|---------|------|
| Headline (`<h3>`, 24px semibold) | No goals in {Month Year}. |
| Body (16px regular, `text-muted-foreground`) | You didn't have goals tracked this month. |
| Back button (`variant="outline"`) | Back to current month |
| Back button link target | `/dashboard/[current-month]` — server computes `formatMonthSegment(monthBucket(now, userTz))` at render |

No Welcome prompt on past-empty. No Copy-from-last-month offer on past-empty (copying into a past month is forbidden anyway — past is immutable per D-21 server enforcement).

### Reflection Card (POLSH-04 + D-24..D-30)

| Element | Copy |
|---------|------|
| Card title (`<h3>`, 24px semibold) | Reflection — {Month Year} |
| "What worked" textarea label | What worked |
| "What worked" textarea placeholder | One thing that went right this month… |
| "What didn't" textarea label | What didn't |
| "What didn't" textarea placeholder | One thing to change next month… |
| Character counter format | `{n}/280` — right-aligned below each textarea, `tabular-nums` |
| Counter colors | 0–249: `text-muted-foreground`. 250–279: `text-warning-foreground bg-warning-muted` (inline `<span>` wrapping the counter text). 280: `text-destructive`. >280: input is clamped at the textarea level (`maxLength={280}`) — the state should not be reachable; if it is (paste overflow), autosave is hard-blocked server-side and inline `<Alert>` surfaces. |
| Save state: autosaving | (nothing visible — silent) |
| Save state: just saved | Saved — quiet 14px semibold, `text-success-foreground` fade-out over 1500ms (CSS transition, no Motion) |
| Save state: save failed | Inline `<Alert variant="destructive">` below the offending textarea with copy "Couldn't save reflection. Please try again." |
| Empty card (no reflection saved yet) | Both textareas render in their placeholder state; no "Saved" indicator; no "Empty" label |

Reflection Card is **not rendered** on future-month views (D-28). It IS rendered on both current and past months, and it is **always editable** (D-27 — no lock after month-end).

### Read-Only Past-Month — No New Copy Required

Past-month cards reuse Phase 2's copy verbatim (raw-count label, progress-bar aria, task labels, habit-cell aria-labels). The difference is **affordance removal**, not copy swap. No "Read-only" banner, no lock icon, no "This month is closed" copy anywhere — PITFALLS §1 portfolio framing is explicit.

### Error / Rollback Copy

| Condition | Copy |
|-----------|------|
| `copyGoalsFromLastMonthAction` returns `{ok: false}` | Couldn't copy last month's goals. Please try again. |
| Reflection autosave returns `{ok: false}` | Couldn't save reflection. Please try again. |
| Reflection Zod validation fails (>280 chars server-side) | That's a bit long — try trimming it to under 280 characters. |
| Past-month mutation attempt (D-12 defense in depth — should never fire from UI) | The toast copy is "This month is archived." — this serves as a developer/debugger signal; end users shouldn't encounter it unless they're replaying network requests. Keep the copy unpunitive: "archived" frames past as a library, not a lockout. |

### Month Nav — Keyboard Shortcut Hints

No visible keyboard-shortcut hints in Phase 3 (they clutter the nav). Advanced users discover `←` / `→` on their own. If Phase 4 adds onboarding, the hints land there.
