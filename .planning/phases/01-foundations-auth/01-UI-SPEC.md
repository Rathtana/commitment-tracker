|------|------|--------|-------------|------|-------|

## Interaction Contract

### Focus

- All interactive elements use `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2` (emerald-400 ring, 2px, 2px offset).
- `ring-offset-background` is `--color-background` so the ring lifts clearly off white/zinc-50 in light mode and zinc-950 in dark mode.
- Mouse users: ring hidden (`:focus` without `:focus-visible`). Keyboard users: ring always visible.
- shadcn's `Button` and `Input` ship with `focus-visible:ring` out of the box; confirm this is wired to `--color-ring` in the shadcn component CSS variables (shadcn new-york uses `--ring` CSS var — map `--ring` to `--color-ring` value in globals.css).

### Button States

| State | Visual |
|-------|--------|
| Default | `bg-primary text-primary-foreground` (emerald-600 fill, white text) |
| Hover | `bg-primary-hover` (emerald-500) — 150ms ease transition |
| Active/pressed | `bg-primary-pressed` (emerald-700) — snap, no transition delay |
| Disabled | `opacity-50 cursor-not-allowed` — no color change |
| Loading (submit) | Spinner (lucide `Loader2` with `animate-spin`, 16px) replaces button text; button remains disabled; button width does not collapse |
| Destructive variant | `bg-destructive text-destructive-foreground` — logout button on landing stub uses `variant="outline"` NOT destructive; logout is a neutral action |

Button height: 40px default (`h-10`). All buttons on auth pages use `w-full` (full width of card content area).

Secondary/ghost buttons (e.g. "Back to sign in" as a link-button): `variant="ghost"` with `text-muted-foreground` at 14px. No background fill.

### Form Field States

| State | Border | Label color | Helper/error text |
|-------|--------|------------|-------------------|
| Default | `--color-border` (zinc-200 / zinc-700) | `--color-foreground` | — |
| Focus | emerald-400 ring (2px, via focus-visible) | unchanged | — |
| Error | `border-destructive` (red-600) + `ring-destructive/30` | unchanged | `text-destructive` 13px below field via `<FormMessage>` |
| Disabled | `opacity-50`, `cursor-not-allowed` | `opacity-50` | — |
| Valid (after submit) | resets to default | unchanged | — |

No "success" green border on valid fields — keep form chrome minimal.

### Password Field

- Input `type="password"` by default.
- Show/hide toggle: lucide `Eye` / `EyeOff` icon button, 44x44px, positioned absolutely at right edge of input (inside input border). Toggles `type` between `"password"` and `"text"`.
- Toggle has `aria-label="Show password"` / `"Hide password"` depending on current state.
- Toggle uses `variant="ghost"` button styling, no ring overlap with input ring.

### Submit Loading State

When form is submitting (`isSubmitting === true` from React Hook Form):
1. Button text replaced by `<Loader2 className="h-4 w-4 animate-spin" />` (lucide, 16px).
2. Button `disabled` attribute set.
3. All form inputs `disabled`.
4. Button width held fixed (no layout shift) — use `min-w-[120px]` or set explicit width if needed.
5. No skeleton/shimmer on form fields — only button state changes.

### Inline Error Slot (Server Action Errors)

Below the form fields, above the submit button: a shadcn `<Alert variant="destructive">` block.
- Hidden when no error (`hidden` class or conditional render).
- Contains `<AlertDescription>` with the error copy.
- `role="alert"` is applied by shadcn's `Alert` component automatically.
- `aria-live="polite"` on the container div wrapping the `Alert` ensures screen readers announce it when it appears.

### Redirect Behavior After Auth Actions

| Action | Redirect |
|--------|---------|
| Successful signup | Stay on `/signup` — show "Check your email to verify your account." inline confirmation. No redirect. |
| Successful login (verified user) | `redirect("/")` (post-login landing stub) |
| Login attempt (unverified user) | Stay on `/login` — show inline Alert: "Please verify your email before signing in. Check your inbox for the verification link." |
| Logout | `redirect("/login")` |
| Successful password reset request | Stay on `/auth/reset` — show inline confirmation: "If that email is registered, you'll receive a reset link shortly." |
| Email verification link clicked → `/auth/callback?code=...` | `exchangeCodeForSession(code)` → `redirect("/auth/verify")` on success, `redirect("/auth/error")` on failure |
| Password reset link clicked → `/auth/callback?code=...&next=/auth/reset/complete` | `exchangeCodeForSession(code)` → `redirect("/auth/reset/complete")` |
| Successful password update | `redirect("/login")` + show `?reset=success` query param so login page shows one-time success message |

### Keyboard Navigation

- Tab order follows DOM order within each form (no tabindex overrides).
- Enter key submits the form when focus is on any field or the submit button.
- Escape key: no special behavior on auth pages (no modals or dialogs in Phase 1).
- All interactive elements reachable by keyboard.

### Semantic Landmarks

Each page must have:
- `<main>` landmark wrapping the auth card content.
- `<h1>` for the page title (see per-surface copy).
- No `<nav>` in Phase 1 auth pages (no navigation menu).
- Landing stub page has `<main>` and a `<header>` containing the logout button.

- No header bar, no footer, no sidebar on auth pages. Single column, full focus on the form.

## Per-Surface Contract

### Surface 1: Signup Page (`/signup`)

**Layout:** Auth card, max-w-sm.

**Copy:**

| Element | Copy |
|---------|------|
| App wordmark | Commitment Tracker |
| h1 (CardTitle) | Create your account |
| Email label | Email |
| Email placeholder | you@example.com |
| Password label | Password |
| Password placeholder | 8+ characters |
| Confirm password label | Confirm password |
| Confirm password placeholder | Repeat your password |
| Submit button | Create account |
| Loading button | (Loader2 spinner — no text) |
| Footer link | Already have an account? Sign in |

**Hidden field:** `<input type="hidden" name="timezone" />` — value populated client-side via `useEffect`: `document.querySelector('[name=timezone]').value = Intl.DateTimeFormat().resolvedOptions().timeZone ?? 'UTC'`. Not shown to user.

**Validation (Zod, client + server):**
- `email`: `z.string().email("Enter a valid email address")`
- `password`: `z.string().min(8, "Password must be at least 8 characters")`
- `confirmPassword`: must match `password` — `z.string()` + `.refine` on the object: `"Passwords do not match"`
- `timezone`: `z.string().default("UTC")` — server-side only; never shown in UI error

**Error messages:**
- Email invalid: "Enter a valid email address"
- Password too short: "Password must be at least 8 characters"
- Passwords mismatch: "Passwords do not match" (on the confirmPassword field)
- Email already registered (server action error, inline Alert): "An account with that email already exists. Sign in instead."
- Generic server error (inline Alert): "Something went wrong. Please try again."

**States:**
- Default: empty form, no errors.
- Submitting: button shows spinner, all fields disabled.
- Success: form resets; inline confirmation message replaces the Alert slot (not an error Alert — use `variant="default"` or a plain div): "Check your email to verify your account. You can close this tab."
- Error: inline Alert with destructive variant above submit button.

**Success behavior:** Stay on `/signup`. Show confirmation copy. No redirect.
